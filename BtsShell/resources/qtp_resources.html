... ${qtpkeepopen}=N
[Documentation] this keyword provide menu operation of Fault Management, the supported operation as below:\n
1. CheckAlarm\n
2. UploadAlarm\n
\n
| Input Parameters \ | Man. | Description | \n
| OperName | Yes | the operation name |\n
| ParaName | Yes | the parameter name list which will be used. \ use ":" to split them |\n
| ParaValue | Yes | the parameter value list which will be used . \ use ":" to split them. it will be one-to-one correspondence the \ ParaName |\n
| qtpkeepopen | No | default N |\n
\n
Operation Parameter description:\n
1.CheckAlarm\n
| ParaName | ParaValue |\n
| AlarmType | the type of alarm, 2 types supported: Active,History |\n
| AlarmNumber | the alarm Number |\n
| StartTime | the start time of alarm, only need date, such as 2012-08-12, it is not necessary parameter |\n
| Node | the object of alarm |\n
| AlarmText | the alarm content |\n
| WaitTime | the largest time user want to wait for the alarm, base on minute,default is 0.5 minute |\n
\n
2. UploadAlarm\n
| ParaName | ParaValue |\n
| Node | target node list |\n
\n
Note:\n
\ \ \ all parameters must be fill in paralist,if you want to use default value, the parameter value could fill ${Empty}\n
\n
\n

\n

| QTP ApplicationLauncher FaultManagement Operation | CheckAlarm | AlarmType:AlarmNumber:StartTime:Node:AlarmText:WaitTime | History:71058:${Empty}:MRBTS-800:NE O&M CONNECTION:${Empty} |\n
\n
3.UploadAlarm --single node\n
| QTP ApplicationLauncher FaultManagement Operation | UploadAlarm | Node | MRBTS-800 |\n
